# 系统总览

```
第一层：感知系统    → 怎么看世界
第二层：处理系统    → 怎么想问题
第三层：输出系统    → 怎么写文章
```

三层缺一不可：
- 只有感知，没有处理 → 观察敏锐但逻辑散乱
- 只有处理，没有感知 → 逻辑严密但选题平庸
- 只有输出，没有前两层 → 文字流畅但言之无物

**核心判断标准：**
```
1. 读者能带走一个新概念吗？
2. 读者会有"原来如此"的感觉吗？
3. 最后一句话，能让人停顿三秒吗？
```
三个都是，这篇文章合格。

## SKILL 01｜三维坐标定位法

> 同样一条新闻，普通人看到事件，进阶者看到趋势。
> 这个技能训练的是第三种能力：看到结构。

**核心动作：** 遇到任何事件，强制完成三个维度的定位，再开始分析。

### 三个维度

```
维度一：技术 / 权力层
→ 这件事改变了什么力量对比？
→ 谁的权力在增加，谁的在减少？
→ 有没有新的技术变量介入？

维度二：制度 / 财政层
→ 现有的制度结构能承接这个变化吗？
→ 财政上是否可持续？
→ 制度的反应速度跟得上变化速度吗？

维度三：历史时间轴
→ 这是第几次出现这种模式？
→ 当下处于这个周期的哪个阶段？
→ 距离上一个临界点有多远？
```

### 操作模板

| 事件 | 技术/权力层 | 制度/财政层 | 历史时间轴 | 共同指向 |
|------|-----------|-----------|----------|---------|
| 填入事件 | 力量对比变化 | 结构能否承接 | 第几次/哪阶段 | 一句话结论 |

### 案例

**中国新能源车出海：**

| 维度 | 分析 |
|------|------|
| 技术/权力层 | 中国掌握电池产业链核心，欧美传统车企技术路线被动跟随 |
| 制度/财政层 | 欧美加征关税，制度层在抵抗，但成本优势短期无法被抹平 |
| 历史时间轴 | 类似日本汽车1970s出海，外部危机触发需求转移是关键变量 |
| 共同指向 | 这是产业权力转移的早期阶段，制度阻力会延缓但无法阻止 |

### 常见错误
- 只填技术层，跳过制度层 → 高估变化速度，低估阻力
- 历史时间轴只写"历史上发生过"→ 必须写当下处于哪个阶段
- 三个维度孤立填写，没有最后那句共同指向的结论

### 练习
每日选一条财经或科技新闻，强制填写三维定位表。坚持30天，定位动作变成条件反射。

## SKILL 02｜因果方向检验

> 大多数观点的错误，不在于事实错误，而在于因果方向错误。

**核心动作：** 主流叙事把因果搞反了吗？

### 四步操作

```
第一步：找到主流叙事的因果链 → A 导致了 B
第二步：反转，重新检验 → B 是否反而导致了 A？
第三步：找历史先例，验证哪个方向更成立
第四步：反转成立 → 这就是你的核心观点
```

### 案例库

| 主流叙事 | 因果反转 | 验证结果 |
|---------|---------|---------|
| 失去霍尔木兹 → 帝国衰落 | 帝国先衰落 → 才失去霍尔木兹 | 反转成立，财政先崩 |
| AI颠覆中国软件股 | 中国SaaS未曾开始 → 何谈颠覆 | 部分成立，前提不同 |
| 老龄化导致消费萎缩 | 财富在老人手里 → 市场没转身 | 反转成立，需求被压制 |
| 高业绩驱动高股价 | 预期先于业绩定价 → 业绩只是验证 | 反转成立 |

### 使用边界
不是所有叙事都值得反转。检验标准只有一个：哪个方向有更多历史先例支撑？

### 练习
每周选一个热点叙事，完整走一遍四步操作。重点在第三步：必须找历史先例，不能只靠逻辑推演。

## SKILL 03｜临界点雷达

> 不追趋势，找临界点。趋势让人麻木，临界点让人警醒。

**核心动作：** 把趋势描述转化为临界点标记。

```
❌ "债务越来越多"         → 趋势，无冲击力
✅ "净利息支出首次超过国防支出" → 临界点，读者会停下来
```

### 三类临界点

```
类型一：量变引发质变
→ 某个比率越过某条线，系统行为发生根本改变
→ 例：利息/军费比越过1，帝国进入被动收缩模式

类型二：两条曲线的交叉点
→ 两个变量此消彼长，交叉那一刻是拐点
→ 例：老年人口规模与供给适配能力的交叉

类型三：不可逆的结构性转变
→ 一旦越过，无法回头
→ 例：制造业外迁之后的产业空心化
```

### 识别提问
```
"什么时候，这个趋势会从量变变成质变？"
"那个节点，有没有一个具体数字或事件可以标记它？"
"越过这个点之后，系统的行为逻辑会发生什么根本改变？"
```

### 练习
对任何议题，强制找出三个潜在临界点。找不到三个，说明理解还停留在趋势层面。

## SKILL 04｜叙事定价审计

> 任何主流叙事背后，都有一个受益方。
> 找到受益方，才能判断这个叙事是分析还是营销。

**核心动作：** 在接受任何叙事之前，先审计它的利益结构。

### 四步操作

```
第一步：找到主流叙事 → "老龄化是社会负担"
第二步：谁在传播？ → 保险公司、养老地产、医疗器械、政策制定者
第三步：传播对他们有什么好处？ → "危机"叙事推动资源倾斜和产品销售
第四步：叙事遮蔽了什么？ → 老人有钱有闲，是被低估的消费主力
```

### 案例库

| 叙事 | 传播方 | 获益方式 | 被遮蔽的事实 |
|------|--------|---------|------------|
| 老龄化是社会负担 | 保险/医疗/政策 | 资源倾斜、产品销售 | 老人持有30%储蓄，是消费主力 |
| AI将替代所有工作 | 科技公司/媒体 | 估值提升、流量获取 | 替代的是任务不是职业 |
| 中国经济硬着陆 | 做空机构/西方媒体 | 仓位获益、地缘叙事 | 制造业转型正在发生 |

### 使用边界
审计叙事不等于否定叙事。审计的目的是：接受叙事之前，先知道它的立场。

### 练习
每周选一个财经热点叙事走一遍四步。重点在第四步：被遮蔽的事实，往往是下一篇文章的选题。

## SKILL 05｜历史数据库检索

> 遇到新问题，自动触发历史检索。
> 历史不是装饰，是验证当下判断的数据库。

**核心动作：** 遇到任何模式，先检索历史上出现过几次。

### 检索模板

```
这个模式，历史上出现过几次？
最相似的案例是哪个？
当时的触发条件是什么？结果是什么？
这次有哪些变量相同，哪些不同？
不同的变量，会让结果往哪个方向偏移？
```

### 核心历史数据库

| 模式类型 | 核心案例 | 关键机制 | 结果规律 |
|---------|---------|---------|---------|
| 技术革命引发失业恐慌 | 五轮技术革命 | 短期替代，长期创造 | 阵痛期由当代人承担 |
| 帝国财政危机 | 英国战后、罗马晚期 | 利息吞噬军费 | 主动或被动收缩 |
| 新兴国家产业出海 | 日本汽车1970s | 外部危机触发需求转移 | 占领对手腹地 |
| 超老龄社会转型 | 日本1990s至今 | 制度适配速度决定结果 | 失去的N年或新物种 |
| 技术股估值重构 | 互联网泡沫2000 | 预期先行，业绩滞后 | 叙事崩塌快于基本面 |
| 大宗品与定制品分化 | 存储vs台积电 | 标准化程度决定定价权 | 大宗品估值天花板低 |

### 推荐书单
- 达里奥《变化中的世界秩序》→ 帝国兴衰的财政-权力模型
- 佩蕾丝《技术革命与金融资本》→ 技术浪潮的周期规律
- 弗格森《文明》《帝国》→ 财政结构与权力边界
- 金德尔伯格《疯狂、恐慌与崩溃》→ 金融危机的重复模式
- 大前研一《低欲望社会》→ 超老龄社会的消费行为变迁

### 建库方法
不要读新闻式的历史，要读机制式的历史。每读完一本书，提炼一个模式填入数据库。数据库越大，当下事件的识别速度越快。

## SKILL 06｜类比解剖法

> 用一个所有人都懂的行业，解释一个大家看不清的问题。

**核心动作：** 识别抽象问题的核心矛盾，找一个具体行业，用它的答案反推抽象问题的答案。

### 好类比的三个标准
- 读者对这个行业有直觉认知
- 这个行业的底层机制和目标问题的底层机制相同
- 类比能推导出一个具体结论，而不只是制造感觉

### 四步操作

```
第一步：识别抽象问题的核心矛盾
        → AI替代什么？核心矛盾：标准化 vs 非标准化

第二步：找一个具体行业，这个矛盾在里面有清晰答案
        → 汽车行业：轮胎/玻璃（标准化）vs 座椅/车灯（定制化）

第三步：用这个行业的答案，反推抽象问题的答案
        → 流程高度耦合、无法模块化的东西，AI也难以替代

第四步：用数据或案例验证推论
        → 存储（大宗品，PE<10）vs 台积电（高定制，PE>20）
```

### 常用类比库

| 抽象问题 | 类比行业/案例 | 核心洞察 |
|---------|-------------|---------|
| 什么有真正的护城河 | 轮胎/玻璃 vs 座椅/车灯 | 流程耦合度决定壁垒高度 |
| 需求确定性如何判断 | 可口可乐 vs 苹果手机 | 消耗品 vs 耐用品估值逻辑不同 |
| 大宗品如何识别 | 存储 vs 台积电 | 标准化程度决定定价权上限 |
| 信任的商业价值 | 老年消费 vs 年轻消费 | 信任不能被算法替代 |

### 练习
每周选一个抽象议题，强制找一个具体行业类比。检验标准：类比能不能推导出一个可以被验证的具体结论？

## SKILL 07｜机制链条构建

> 不描述现象，推演机制。能写出链条，才算真正想清楚了。

**核心动作：** 把"A导致B"扩展成至少4步的完整因果链。

### 现象思维 vs 机制思维

```
现象思维：债务多了，美国会衰落。

机制思维：
利息吞噬财政
→ 军费空间被挤压
→ 霸权维护能力下降
→ 盟友信心动摇
→ 主权信用重定价
→ 融资成本抬升
→ 利息更重
→ 自我强化的负循环
```

### 五步构建法

```
第一步：找到起点（已发生的、可验证的事件）
第二步：问"这会直接导致什么？"→ 第一阶效应，通常显而易见
第三步：对第一阶效应重复提问 → 二阶、三阶效应，通常是别人没想到的
第四步：找到自我强化或自我终止的节点
        → 负循环：越来越差 / 正循环：越来越好 / 终止点：什么条件下链条断裂？
第五步：用历史先例验证这条链条跑通过吗？
```

### 链条质量标准

```
✅ 合格：至少4步 / 每步有机制支撑 / 末端能解释当下或预测未来 / 有历史先例验证
❌ 不合格：少于3步 / 中间有跳步靠直觉填充 / 只有方向没有机制 / 无法被历史验证
```

### 练习
对任何趋势，强制写出4步以上机制链条。每一步问自己：为什么A会导致B，而不只是"然后"？

## SKILL 08｜框架命名法

> 有名字的东西才能被记住，才能被传播，才能成为别人思考的工具。

**核心动作：** 发现一个规律，给它起名字。这是把分析转化为观点的关键动作。

### 三种命名方式

```
方式一：借用已有框架，精准引用
→ 弗格森点、范式速度差、技术-经济范式
→ 要求：真正理解这个框架，不是借名字装门面

方式二：自己提炼，用"X的Y"结构
→ "帝国的财政临界点" / "制度的速度债" / "信任的护城河"

方式三：用一句话定义两个概念的关系
→ "军费是为未来花钱，利息是为过去还债"
→ "轮胎是流程，座椅是模块"
→ "股价是公司，也是他者的预期"
```

### 检验标准
> 能不能用一句话，让一个没有背景知识的人理解这个概念？
> 能 → 你真的想透了。不能 → 你在用名字掩盖模糊的理解。

### 练习
每周读完一篇深度文章或一本书，提炼一个规律，给它命名。用"X的Y"结构尝试，再用一句话解释它。

## SKILL 09｜悬念结构设计

> 普通文章：给结论 → 给论据。
> 悬念结构：给论据 → 让读者下注 → 颠覆预期 → 才给结论。

**适用场景：** 反直觉结论 / 投资决策类 / 需要读者主动参与的议题

### 五步悬念结构

```
第一步：布局 → 给出所有支持"显而易见"结论的证据，让读者觉得答案已明了
第二步：钩子 → 问读者：你会怎么选？你心动了吗？让读者在心里下注
第三步：第一次反转 → 揭示出乎意料的结果，冲击力来自读者已经下注
第四步：第二次反转（可选）→ 再给一个反例，彻底打破预设框架
第五步：收网 → 才给出真正的论点，此时读者已被充分激活
```

### 案例示范（泡泡玛特）

```
布局：利润+127%、+88%，预测+308%，PE只有14倍，毛利率70%
钩子：你心动了吗？
反转：买入后5天浮亏35%
二次反转：精准预测到疫情，反而迎来流动性泡沫行情
收网：重要的从来不只是何事发生，而是人们如何回应它
```

## SKILL 10｜八步成文结构

> 把思考转化为文章的标准流程。
> 每一步都有明确功能，缺少任何一步，文章就会有明显短板。

### 八步结构

**① 开篇锚点**
功能：建立立场，制造"这个人不一样"的第一印象
标准：读完第一句，读者知道这篇文章的核心判断是什么
禁忌：不能以背景介绍开头，不能以"随着XX的发展"开头

**② 引入权威/事件**
功能：借力，降低读者门槛，为反转做铺垫
标准：读者感到熟悉，但预感要被颠覆
技巧：引入的权威或事件，必须是你准备反驳或深化的对象

**③ 反转或深化**
功能：确立作者视角，建立可信度
标准：读者感到"这个人有自己的判断"
格式参考："但这个逻辑反了。" / "这个判断只对了一半。"
要求：必须紧跟论据，不能只是态度

**④ 硬框架登场**
功能：给读者一个可以带走的新概念
标准：有名字，有定义，能用一句话解释清楚
来源：已有理论的精准引用，或自己提炼的命名

**⑤ 数字或案例落地**
功能：让框架无可辩驳
标准：具体到让人无法说"这只是理论"
原则：宁可少，不能假。没有数字就用具体案例替代

**⑥ 机制链条**
功能：展示思维深度，给读者完整的因果闭环
标准：至少4步，每步有机制支撑，末端能解释当下或预测未来
效果：读者跟着链条走完，感到"原来如此"

**⑦ 当下困境印证**
功能：用框架解释一个正在发生的现象
标准：读者感到框架有预测力，不是马后炮
技巧：选一个读者最近刚看到的新闻或现象作为印证

**⑧ 落回人的处境**
功能：把宏观结论翻译成具体的人，制造情感共鸣
标准：读者感到"这说的是我"或"这说的是我们这代人"
最后一句：制造停顿感，让读者沉默三秒
禁忌：不用"总结""综上"，直接说那句最重的话

---

## SKILL 11｜语言四原则

> 把"说清楚了"变成"写好了"的最后一步。

### 原则一：数字替代形容词

```
❌ 美国债务压力非常大
✅ 净利息支出8799亿，首次超过国防支出8735亿，差额64亿

❌ 老年人口规模庞大
✅ 60岁以上人口超过2亿，持有全国约30%的储蓄存款
```

### 原则二：机制替代判断

```
❌ 这对行业很不利
✅ 成本上升 → 利润压缩 → 研发投入减少 → 竞争力下降 → 市场份额流失

❌ 存储芯片估值低是有原因的
✅ 标准化程度高 → 无差异化竞争 → 定价权在买方 → 利润随周期波动 → 市场给大宗品估值
```

### 原则三：人替代概念

```
❌ 市场情绪发生了变化
✅ 那些和你一样在买、在卖、在犹豫、在等待的人，集体改变了预期

❌ 阵痛期不可避免
✅ 阵痛期不是没有代价的，它由刚好活在错误年份里的那一代人承担
```

### 原则四：短句收尾，制造停顿

```
❌ 因此我们可以得出结论，这个时代的结构性机会已经悄然转移
✅ 机会转移了。

❌ 综上所述，银发经济的真正价值还远未被市场充分认识
✅ 钱在那里，缺的只是一个认真转过身来的人。
```

### 风格禁区
以下内容一律不写：
- 开头是"随着XX的快速发展"
- 结尾是"希望对大家有所帮助"
- 文中出现"笔者认为""综上所述"
- 用"非常""极其""十分"修饰任何判断
- 给出结论但不给机制
- 给出现象但不给因果
- 历史先例缺失，纯靠逻辑推演

---

## SKILL 12｜标题三型法

> 标题是文章的第一个观点，不是文章的目录。

### 类型一：反转型
颠覆主流认知的判断
结构：[主流认知的反面] + [具体对象]

示范：
- "钱在老人那里，眼睛却盯着年轻人"
- "不是AI太强，是中国SaaS从未真正开始"
- "美国SaaS估值先崩了，中国这边反而稳住"

### 类型二：临界点型
点出量变到质变的节点
结构：[具体节点] + [系统性变化]

示范：
- "利息超过军费的那一年，帝国进入倒计时"
- "当储蓄开始流动，银发经济才真正开始"
- "PE跌破10的那一刻，市场在说什么"

### 类型三：人的处境型
把宏观落回到具体的人
结构：[宏观变化] + [对具体人的意义]

示范：
- "刚好活在错误年份里的那一代人"
- "做时间的朋友，但保持敬畏"
- "那些和你一样在等待的人"

### 标题禁区
```
❌ 浅析XX的发展现状与未来趋势
❌ 关于XX问题的几点思考
❌ XX行业，你了解多少？
❌ 震惊！XX竟然是这样的
```

---

## SKILL 13｜自检清单

> 发布前的最后一道关卡。
> 不是检查措辞，是检查思维是否完整。

### 结构自检

```
□ 开篇第一句，有明确的核心判断吗？
□ 文章有反转或深化，而不只是陈述吗？
□ 有一个可以带走的命名框架吗？
□ 有具体数字或案例支撑框架吗？
□ 机制链条至少4步吗？
□ 有当下现象印证框架吗？
□ 结尾落回到具体的人了吗？
□ 最后一句话，能让人停顿三秒吗？
```

### 语言自检

```
□ 有没有用形容词替代数字？→ 全部换成数字
□ 有没有用判断替代机制？→ 全部展开成链条
□ 有没有用概念替代人？→ 全部换成具体的人
□ 结尾是长句吗？→ 改成短句
□ 出现"综上""笔者""随着"了吗？→ 全部删除
```

### 思维自检

```
□ 核心观点，是反直觉的吗？如果不是，值得写吗？
□ 有历史先例支撑吗？还是纯靠逻辑推演？
□ 因果方向检验过了吗？有没有把因果搞反？
□ 读者能带走一个新概念吗？
□ 读者会有"原来如此"的感觉吗？
□ 最后一句话，能让人停顿三秒吗？
```

最后三个问题，三个都是 → 发布。有一个不是 → 重写对应部分，不是修改措辞。

# 示范文章：银发经济

> 以下是用这套系统完整走一遍后产出的文章示范。
> 括号内标注了对应的Skills。

---

**【开篇锚点 · SKILL 10①】**

钱在老人那里，眼睛却还盯着年轻人。

---

**【引入权威/事件 · SKILL 10②】**

2023年，中国60岁以上人口突破2.97亿，占总人口的21.1%。所有人都在谈老龄化危机：劳动力萎缩、养老金缺口、医疗支出膨胀。这是主流叙事，也是真实的压力。

---

**【反转 · SKILL 02 + SKILL 10③】**

但这个叙事反了一半。

老龄化是负担，这是制度层的判断。但从消费层看，老人不是负担，是被低估的主力。中国60岁以上人口持有全国约30%的储蓄存款。他们有钱，有闲，有需求——缺的不是购买力，是一个认真转过身来对着他们的市场。

---

**【硬框架登场 · SKILL 08 + SKILL 10④】**

这里有一个值得命名的现象：**制度的速度债**。

工业时代建立的制度——退休年龄、养老体系、消费品类——是为一个劳动力充裕、老人稀少的社会设计的。当人口结构反转，制度没有同步反转。制度欠了一笔速度债，而这笔债的利息，由那些活在制度与现实错位之间的人来承担。

---

**【数字落地 · SKILL 11 + SKILL 10⑤】**

具体到消费：
- 60岁以上人口持有约30%储蓄，但针对他们的消费品类占零售总额不足5%
- 日本银发经济市场规模超过100万亿日元，中国同等规模市场尚未形成
- 老年人复购率高于年轻人，但品牌投入的营销预算中，针对老年用户的不足10%

供给和需求之间，有一条巨大的裂缝。

---

**【机制链条 · SKILL 07 + SKILL 10⑥】**

为什么这条裂缝还没有被填上？

```
品牌认知停留在"老人没钱"的旧叙事
→ 营销预算向年轻用户倾斜
→ 老年用户需求得不到精准供给
→ 老年用户消费意愿被压制
→ 数据显示老年消费低迷
→ 品牌用数据验证了旧叙事
→ 自我强化的认知循环
```

这不是市场失灵，是认知滞后。

---

**【当下困境印证 · SKILL 10⑦】**

日本走过了这条路。1990年代，日本老龄化加速，最先转身的不是大品牌，是便利店。7-Eleven开始在门店增加适合老年人的小份装、低糖食品、助行器租赁点。结果是：老年客群成为便利店最稳定的消费群体，复购率和客单价都高于年轻用户。

转身早的人，先吃到了这块蛋糕。

---

**【落回人的处境 · SKILL 10⑧】**

那些在制度和市场的双重忽视里老去的人，并不是没有需求。他们只是习惯了被忽视。

钱在那里。缺的只是一个认真转过身来的人。
